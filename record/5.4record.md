## WREC expert transfer microbenchmark

### 本次操作

- 新增 `workspace/scripts/wrec/benchmark_expert_transfer.py`。
- 使用 Mixtral-8x7B-Instruct-v0.1 config 推断单 expert 大小，并在当前 GPU 上测量 host-to-GPU copy 时间。
- 输出：
  - `workspace/results/wrec/calibration/wrec_expert_transfer_mixtral8x7b_20260504.json`
  - `workspace/results/wrec/calibration/wrec_expert_transfer_mixtral8x7b_20260504.md`
- 更新 `workspace/notes/wrec_experiment_plan_20260427.md` 的 Phase 7 校准状态。
- 更新 `workspace/results/wrec/phase8/wrec_phase8_paper_ready_summary_20260503.md` 的硬件校准证据。

### 操作原理

- WREC replay 的 stall proxy 来自 `expert_bytes / bandwidth`。
- 因此需要用真实硬件测量单 expert 大小 tensor 的 CPU/host memory 到 GPU copy time，验证 `41.37 GB/s` 带宽假设是否与当前机器一致。
- 同时分别测量 pageable 与 pinned host memory，区分普通 host memory copy 与适合 offload/prefetch 的 pinned staging。

### 结果

- Expert bytes: `352321536`，即 `336.00 MiB`。
- Simulator assumed transfer: `8.516353 ms/expert` at `41.37 GB/s`。
- Repeats / warmups: `100 / 5`。
- Pageable host memory:
  - median `23.455377 ms/expert`
  - median `15.020929 GB/s`
- Pinned host memory:
  - median `8.467264 ms/expert`
  - median `41.609846 GB/s`

### 结论

- pinned memory 实测传输时间与 simulator 使用的 transfer time 基本一致，支持当前 WREC replay 的硬件校准口径。
- pageable memory 明显更慢，后续 runtime 落地讨论应明确 pinned host staging 假设。
- 该结果仍是 transfer microbenchmark，不是端到端 serving latency。

### 操作原理

- shadow mode 不控制真实 expert loading，只按 routed expert event 到达顺序在线更新 WREC state，并维护一个 shadow resident set。
- 每个 expert ref 到达后立即判断 shadow hit/miss；miss 时立即执行 WREC admit/bypass/evict 决策。
- 该流程不读取 future trace，用于验证 WREC policy/state 是否符合 runtime-compatible event contract。
- 使用同一 eval trace、train prior、cache budget 和权重配置，与 total-budget replay 做行为对齐。
- 对齐验证脚本读取 shadow JSON 和 total-budget replay CSV，逐项比较 expert refs、hit/miss count、hit/miss rate、transfer bytes 和 stall metrics。

- Shadow misses: `147884`。
- Shadow miss rate: `0.16333409910228316`。
- Total-budget replay miss rate: `0.16333409910228316`。
- Shadow stall per input token: `89.01995013339396 ms`。
- Total-budget replay stall per input token: `89.01995013339396 ms`。
- Would-admit: `112416`。

- 该结果降低了“WREC 只是离线脚本”的风险，说明其 policy/state 可以包装成 runtime-compatible shadow component。
- 该模式仍不控制真实 expert loading，不证明端到端 serving latency 改善。
