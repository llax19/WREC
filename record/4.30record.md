## GPU0 占用排查

### 本次操作

- 执行 `nvidia-smi` 查看 GPU0/GPU1 显存与标准进程表。
- 执行 `nvidia-smi --query-compute-apps=gpu_uuid,pid,process_name,used_memory --format=csv,noheader,nounits` 查看驱动记录的 compute app。
- 执行 `python scripts/runtime/scan_nvidia_users.py` 扫描当前容器 `/proc/*/fd` 中是否有进程打开 `/dev/nvidia*`。
- 检查 `PID 2854639` 是否存在于当前容器 `/proc`。
- 执行 `nvidia-smi pmon -c 1` 辅助确认当前采样时是否有可见活跃进程。

### 操作原理

- `nvidia-smi` 的显存用量来自 NVIDIA driver，全局反映 GPU 状态；但进程表在容器 PID namespace 下可能无法解析宿主机或其他容器中的 PID。
- 如果占用来自当前容器，通常可以在当前容器 `/proc` 中找到对应进程，或者通过扫描 `/proc/*/fd` 发现进程持有 `/dev/nvidia0`、`/dev/nvidiactl`、`/dev/nvidia-uvm` 等设备文件。
- 如果 driver 能给出 PID，但当前容器没有该 PID，且容器内设备 fd 扫描也没有 holder，则大概率是宿主机或其他容器进程占用。

### 结论

- GPU0 当前显示约 `21873MiB / 24564MiB` 被占用，GPU1 约 `10MiB / 24564MiB`。
- 标准 `nvidia-smi` 进程表为空，但 compute app 查询显示 GPU0 上有 `PID 2854639` 使用约 `21844MiB`，进程名为 `[Not Found]`。
- `PID 2854639` 不存在于当前容器 `/proc`。
- 当前容器内未发现可见进程持有 `/dev/nvidia0`。
- 当前证据表明 GPU0 占用不是本容器内可见进程造成的，更可能来自宿主机或其他容器。

## Mixtral Phase 2B n=1 probe

### 本次操作

- 在 `vllm_moe` 环境运行 Mixtral n=1 offload feasibility probe。
- 使用模型目录 `models/Mixtral-8x7B-Instruct-v0.1`。
- 使用 request manifest `data/prompts/wrec_dolly_debug_n64.jsonl`。
- 设置 `--limit 1`、`--max-input-tokens 128`、`--dtype float16`、`--device-map auto`。
- 设置保守显存约束 `--max-memory 0=22GiB,1=22GiB,cpu=96GiB`，并指定 offload 目录 `models/offload/mixtral8x7b`。
- 运行期间监控 GPU 显存、CPU 内存和 Python 进程 RSS。
- 失败后检查 trace/stats/failure 输出文件、GPU 残留状态和当前容器 `/proc` 可见进程。
- 新增失败记录文件 `results/wrec/mixtral8x7b_offload_probe_failure_20260430.json`。

### 操作原理

- Phase 2B 的目的不是先做完整实验，而是用 1 条短 request 验证 Mixtral 权重、Transformers/Accelerate 加载、CPU/GPU offload、prefill forward 和 router event 提取链路。
- `max_input_tokens=128` 用来把 prompt 计算压力压到最低，使失败原因尽量集中在模型加载/offload 可行性本身。
- `max_memory` 用来限制 Accelerate 的 device map，避免单卡直接吃满显存；但 Transformers 在 Mixtral expert 权重自动转换/finalization 时仍需要额外临时显存。

### 结论

- 本次 n=1 probe 失败，且失败发生在 `AutoModelForCausalLM.from_pretrained` 的权重加载/转换阶段，尚未进入 request prefill，也没有生成 router event trace。
- 权重加载进度到达 `291/291` 后，Transformers 在 Mixtral expert `gate_up_proj` 自动转换/finalization 时触发 GPU1 OOM。
- 报错显示 GPU1 总容量约 `23.68GiB`，仅剩约 `29.94MiB`，额外分配 `896MiB` 到 `1.75GiB` 失败。
- 运行中观测到 GPU0 约 `23002MiB`、GPU1 约 `24217MiB`，Python RSS 最高约 `112781MiB`。
- 本次没有生成 `logs/processed/wrec/mixtral8x7b_dolly_probe_router_events_n1_20260430.jsonl`、`results/wrec/mixtral8x7b_offload_probe_n1_20260430.json` 或 request-level failure jsonl，因为失败发生在模型加载完成前。

- 当前不应立刻重试；需要先等待或由宿主机侧清理 GPU0 残留占用。
- 下一次建议在 GPU0 完全释放后改用更低 GPU 上限，例如 `0=20GiB,1=20GiB,cpu=96GiB`，并加 `PYTORCH_ALLOC_CONF=expandable_segments:True`，给 Transformers 加载/转换阶段留出额外显存余量。
