\section{实验设计与结果分析}

本章评估 WREC 在资源受限 MoE 预填充专家缓存场景中的效果。实验围绕四个方面展开：首先验证目标模型与硬件配置下的专家传输压力；其次分析预填充路由轨迹中的专家访问局部性；随后比较 WREC-H2 与在线基线、静态基线和离线上界之间的差异，并通过消融和敏感性实验分析收益来源；最后给出运行时影子与有限槽位原型结果，验证方法接入真实推理系统的可行性。

\subsection{实验平台与数据来源}

本文主实验使用 Mixtral-8x7B-Instruct-v0.1 的预填充路由轨迹。Mixtral 是典型的稀疏 MoE 大语言模型，其路由器在每个 MoE 层中为词元选择少量专家参与计算 \cite{jiang2024mixtral,mixtral,shazeer2017outrageously}。因此，Mixtral 的预填充过程能够提供按层展开的专家访问序列，适合作为本文评估专家缓存策略的主要对象。工作负载来自 Databricks Dolly 15k 指令数据集，该数据集包含开放问答、封闭问答、信息抽取、摘要、生成和分类等任务类型~\cite{databricks2023dolly}。本文将 Dolly 样本划分为训练/参考轨迹和评估轨迹：训练轨迹用于构造静态热度、窗口先验和跨层转移概率；评估轨迹仅用于策略重放评估。Belady 离线上界和路由窗口预取 oracle 使用评估轨迹中的未来窗口，分别作为离线上界和压力基线。

表~\ref{tab:experiment-setup} 总结了主要实验配置。专家大小按 Mixtral 专家 FFN 权重推断，float16/BF16 下单专家对象为 $352321536$ bytes，即约 $336$ MiB。CPU-GPU 有效带宽采用 $41.3722$ GB/s，该值来自专家传输微基准。按该带宽估算，单个专家对象加载时间约为 $8.5164$ ms，与 pinned host memory 到 GPU 的实测中位复制时间 $8.4673$ ms 基本一致。本文后续报告的传输时间估计值用于比较不同策略下的专家传输压力。

\subsection{专家传输瓶颈筛查}

本节首先评估目标配置中的专家传输压力。Mixtral-8x7B 的参数主要集中在专家 FFN 权重中。MoE 模型通常将大量参数放置在专家 FFN 中，推理时只激活其中一部分专家，这也是专家缓存和专家卸载问题产生的主要原因之一 \cite{shazeer2017outrageously,jiang2024mixtral,xue2024moeinfinity,cao2025moelightning}。按 BF16/float16 估算，模型总权重约 $86.99$ GiB，其中专家 FFN 权重约 $84.00$ GiB。在资源受限单机或消费级 GPU 配置下，全部专家常驻会与非专家权重、KV cache 和运行时临时缓冲共同竞争显存空间，因此 partial residency 是该场景下的实际约束。已有显存受限 LLM 推理和 MoE 推理系统进一步表明，专家权重的驻留、卸载和迁移会显著影响端到端推理性能 \cite{sheng2023flexgen,aminabadi2022deepspeed,xue2024moeinfinity,cao2025moelightning}。

为确认本文实验配置是否处于专家传输可能成为瓶颈的区域，本文根据模型权重规模、GPU 显存预算、KV cache 预留量以及实测 CPU--GPU 传输带宽，对不同资源配置进行 HRM bottleneck 判定。该判定不估计各阶段耗时占比，而是判断在给定资源约束下专家权重迁移是否可能成为主导瓶颈。图~\ref{fig:hrm-transfer-bound} 给出了不同显存预算和 KV cache 预留量下的判定结果。

\begin{figure}[!htbp]

    \caption{\label{fig:hrm-transfer-bound}Mixtral-8x7B 在不同 GPU 显存预算与 KV cache 预留量下的 HRM bottleneck 判定。每个单元格表示该配置是否进入 expert-transfer-bound 状态。}

结果显示，在 $24$ GiB 和 $48$ GiB 显存预算下，所有测试配置均被判定为 expert-transfer-bound；在 $96$ GiB 显存预算下，只有当 KV cache 预留量增大到 $16$ GiB 及以上时才进入该状态。这说明在本文关注的受限显存配置中，专家权重无法简单视为常驻资源，专家加载与迁移开销需要在缓存策略评估中被显式计入。因此，后续实验不将 WREC 表述为完整端到端推理系统的加速结果，而是围绕需求专家加载次数、传输字节数和由实测带宽换算得到的传输时间估计值评估其对专家传输压力的缓解效果。
\subsection{路由局部性分析}

专家访问序列的局部性直接影响缓存替换策略的有效性。近期关于 MoE 本地路由一致性的研究指出，不同 MoE 模型在连续词元上的专家选择稳定性差异明显，因而并非所有模型都同样适合专家卸载和缓存优化~\cite{liang2026notallmodels}。对于 MoE 预填充轨迹，关键问题在于专家访问是否呈现稳定的频率集中、短距离复用以及跨层相关性。若访问分布近似均匀且跨数据划分差异较大，基于历史统计或在线上下文的缓存策略难以优于简单替换规则；若热门专家和相邻访问结构较稳定，则有限槽位仍可能覆盖较多后续访问。

表~\ref{tab:locality-statistics} 给出了 Mixtral Dolly 训练轨迹的局部性统计。该轨迹包含 $29123$ 个输入词元单元、$931936$ 个路由事件和 $1863872$ 个专家引用。每个输入词元平均产生 $32$ 个路由事件和 $64$ 个专家引用，与 Mixtral 的 $32$ 层 top-$2$ MoE 结构一致 \cite{jiang2024mixtral,mixtral}。这些基础统计用于确认轨迹展开与模型结构一致，避免后续缓存结果建立在异常采集或错误展开之上。

从统计结果看，同层专家引用的复用距离 p50/p90/p99 为 $5/27/7097$。这说明大多数同层复用发生在较短距离内，但仍存在明显长尾；因此，仅保留最近一次访问并不能完整刻画未来访问，缓存策略需要同时考虑短窗口复用和更稳定的频率先验。另一方面，训练集与评估集的 top-$2$ 和 top-$4$ 热门专家重合度均值均为 $0.9375$，表明热门专家在不同数据划分间较稳定，训练轨迹统计可以作为评估阶段的先验。请求内单层工作集的 p50/p90/p99 均为 $8$，也说明在单个请求和单层范围内，专家集合会迅速覆盖该层全部 $8$ 个专家；这意味着“每层固定少量常驻专家”的策略会受到容量限制，而全局自适应槽位分配更有必要。

因此，Mixtral 预填充路由并非随机均匀访问，而是同时具有短距离复用、跨划分热度稳定性和请求内工作集扩张三种特征。前两者为 WREC-H2 的评分信号提供依据，后者解释了为什么后续实验不能只依赖静态热门专家，而需要在总槽位预算下动态调整驻留集合。

\subsection{公平基线与离线上界}

表~\ref{tab:fair-baselines} 的作用是为后续 WREC 主实验建立参照系。结果显示，专家缓存调度不能由单一简单规则充分解决。容量较小时，静态热度优于 LRU，表明训练轨迹的频率先验具有价值；容量较大时，LRU 略优于静态热度，表明在线时间局部性同样重要。两类简单基线与 Belady 离线上界之间仍存在明显差距：例如每层 $4$ 个槽时，LRU 的传输时间估计值为 $228.3359$ ms/input token，而 Belady 为 $119.5856$ ms/input token。该差距说明，在相同路由轨迹和相同容量约束下，简单在线策略尚未充分利用未来访问结构。后续 WREC-H2 的目标正是在不使用未来信息的前提下，用可在线获得的请求内和跨层信号缩小这一差距。

\subsection{WREC 主结果}

WREC 主实验使用固定总专家缓存预算，而不是固定每层容量。Mixtral 共有 $256$ 个专家对象，因此总槽位数 $32/64/96/128/192$ 分别对应 $12.5\%/25.0\%/37.5\%/50.0\%/75.0\%$ 的专家对象容量。LRU 和路由窗口预取采用均匀逐层分配；\texttt{static\_hot} 使用全局训练频率；WREC 使用全局自适应准入/换出。该设置与现有系统中围绕有限显存预算进行权重驻留、分层缓存或卸载调度的研究问题一致 \cite{sheng2023flexgen,song2023powerinfer,xue2024moeinfinity,cao2025moelightning}。表~\ref{tab:wrec-main-result} 给出主要结果。

WREC-H2 在全部测试预算上均优于 LRU。随着总槽位数从 $32$ 增加到 $128$，WREC 相对 LRU 的传输时间估计值改善从 $71.80\%$ 增至 $81.40\%$；在 $192$ 个槽位时收益回落到 $75.27\%$，但仍保持较高幅度。收益随容量增加先升后降的原因是：中等容量下，LRU 仍会频繁被短期访问扰动，而 WREC-H2 可以利用请求内和跨层信号选择更可能复用的专家；当容量继续增加时，LRU 自身命中率提高，可优化空间缩小，相对收益自然下降。WREC-H2 在所有主配置中的预取浪费均为 $0$，原因是该版本仅执行需求加载后的准入/换出，不依赖未来窗口预取。

图~\ref{fig:wrec-stall-budget} 用连续预算点展示了这一趋势。WREC 在全部预算点上均低于 LRU，同时仍高于 Belady 离线上界。这一结果说明，在不改变模型路由结果的前提下，专家缓存策略本身可以显著影响需求加载次数和传输压力；这一点与 MoE-Infinity、MoE-Lightning 等工作对专家卸载和驻留管理的关注方向一致 \cite{xue2024moeinfinity,cao2025moelightning}。更重要的是，WREC 与 Belady 之间仍存在差距，说明本文方法并不是通过离线未来信息“拟合”轨迹，而是在可部署信息边界内捕获了部分可利用结构。

\subsection{消融实验}

\subsection{运行时接口与有限槽位验证}

前述实验基于访问轨迹重放，能够隔离比较专家缓存策略本身。类似的离线或半离线分析常用于将调度、缓存和内存管理策略从模型计算本身中分离出来 \cite{yu2022orca,kwon2023pagedattention,sheng2023flexgen}。为验证 WREC 的在线运行形式，本文首先实现了运行时影子原型。该原型按路由专家事件流逐条消费事件，在线维护请求内局部性、跨层历史和影子驻留集合，并输出拟准入、拟绕过、拟换出决策。该原型不控制真实专家加载，因此其作用是验证在线状态更新与决策一致性。

在 eval n256 评估轨迹上，影子原型使用 $64$ 总槽位数、历史长度 $8$、最近历史权重 $0$、请求内权重 $1024$、跨层权重 $1024$。结果如表~\ref{tab:runtime-shadow} 所示。影子未命中率为 $0.163476576$，传输时间估计值为 $89.0976$ ms/input token。与同配置离线总槽位预算重放对齐验证中，专家引用、命中/未命中次数、命中/未命中率、传输字节数和时间指标的差异均为 $0$，总体通过结果为 \texttt{True}。

\subsection{有效性威胁与局限性}

本文实验存在以下局限。第一，主结果来自轨迹驱动模拟器，使用实测带宽和专家大小构造专家权重传输时间估计值，尚未完整建模计算与传输重叠、CUDA 调度、内存分配、显存碎片和运行时排队；这些因素在完整大语言模型服务和 MoE 服务系统中会与批处理、并发调度和内存管理共同影响端到端性能 \cite{yu2022orca,kwon2023pagedattention,zheng2024sglang,cao2025moelightning}。第二，路由窗口预取和 Belady 离线上界使用评估轨迹中的未来信息，分别作为压力基线和离线上界，不对应可部署在线策略。第三，当前主实验假设同一模型内专家大小相同，传输感知评分在异构专家大小或异构传输代价下的作用仍需进一步检验。第四，本文主结果覆盖 Mixtral 预填充轨迹；解码迁移、扩展部署分析未作为主文正结果展开。

据此，本文结论限定在专家传输受限、部分专家驻留的 Mixtral 预填充轨迹场景内：WREC-H2 在不改变路由结果的前提下降低需求专家加载次数和传输时间估计值。模型质量、Mixtral 端到端 TTFT、完整服务延迟以及解码阶段效果不在本文主实验的结论范围内。有限槽位实验表明运行时接入路径具备初步可行性，仍需在更完整的专家卸载和服务基准中继续验证。

综合实验结果可以看出，HRM 筛查和传输微基准为专家缓存优化提供了配置依据，路由统计说明预填充阶段轨迹中存在短窗口复用和跨数据划分的热度稳定性，公平基线与 Belady 离线上界之间的差距则表明简单热度缓存不足以完全利用这些结构。在固定总槽位预算主实验中，请求内/跨层 WREC-H2 在 $32$ 到 $192$ 个专家槽上均优于 LRU，传输时间估计值改善为 $71.80\%$ 到 $81.40\%$；消融结果进一步表明，主要收益来自请求内局部性和跨层转移信号，而非最近历史项。运行时影子验证与 Qwen 有限槽位参数扫描说明 WREC 的接入路径具备初步可行性，但本文主结论仍限定在预填充阶段专家缓存调度和轨迹驱动重放结果之内。
